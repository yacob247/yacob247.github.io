4. Python LoRA Script

Low-Rank Adaptation works by freezing base layers and inserting rank decomposition matrices into the attention layers. Mathematically, the update matrix $\Delta W$ is decomposed into two low-rank matrices $B$ and $A$: $$\Delta W = B \cdot A$$ where $B \in \mathbb{R}^{d \times r}$, $A \in \mathbb{R}^{r \times k}$, and the rank $r \ll \min(d, k)$. This reduces trainable parameters by over $99\%$.

Save and run the following script in your Colab terminal to begin a training simulation:

train_lora.py
import torch
from transformers import AutoModelForCausalLM, AutoTokenizer, BitsAndBytesConfig
from peft import LoraConfig, get_peft_model, TaskType

# Configure 4-bit quantization to fit into 16GB VRAM
quant_config = BitsAndBytesConfig(
    load_in_4bit=True,
    bnb_4bit_quant_type="nf4",
    bnb_4bit_compute_dtype=torch.float16
)

model_id = "TinyLlama/TinyLlama-1.1B-intermediate-step-1431k-3T"
model = AutoModelForCausalLM.from_pretrained(model_id, quantization_config=quant_config)
tokenizer = AutoTokenizer.from_pretrained(model_id)

# Define LoRA Configuration
lora_config = LoraConfig(
    r=8, 
    lora_alpha=16,
    target_modules=["q_proj", "v_proj"],
    lora_dropout=0.05,
    bias="none",
    task_type=TaskType.CAUSAL_LM
)

model = get_peft_model(model, lora_config)
model.print_trainable_parameters()
# Output will confirm parameters are scaled down massively!

ðŸ§  Fine-Tuning vs. Foundation Architectures

Ã—

Understanding the fundamental difference in how models like ChatGPT or Gemini are trained compared to running parameter-efficient fine-tuning (PEFT) on your own machine is essential for modern engineers.

1. Base Pre-Training (Gemini, GPT-4, Claude)

Foundation models are trained from scratch using Pre-Training. It is a highly computational, expensive, and multi-month process.

  • Scale: Up to hundreds of billions of parameters.
  • Math: Training updates the entire weight matrix $W$ across every single layer.
  • Dataset: Massive, unlabelled internet corpora (Common Crawl, Wikipedia, Books).
  • Compute cost: Millions of dollars, requiring thousands of clustered H100 GPU units working in synchronous parallel.

2. Local / Cloud Fine-Tuning (LoRA & QLoRA)

Fine-tuning is the process of taking an existing, already intelligent foundation model and modifying its output style or domain knowledge to match your specific context.

Instead of modifying the massive base weight $W_{0}$, LoRA restricts the updates to a separate adapter:

$$W = W_{0} + \Delta W = W_{0} + B \cdot A$$

During training, the base model weights ($W_{0}$) are completely **frozen** (no gradients computed), while $B$ and $A$ are trained on your custom dataset. This has several massive advantages:

  • VRAM savings: Memory required for backpropagation gradients scales only with the small adapter matrix size.
  • No Catastrophic Forgetting: The base model retains its underlying foundational intelligence.
  • Modular Hot-Swapping: You can keep the same base model running in memory and swap small 20MB adapter files depending on the incoming prompt task.

import http from 'http';

async function promptOllama(promptText) {
  const payload = JSON.stringify({
    model: "cyber-sec-ai",
    prompt: promptText,
    stream: true
  });

  const options = {
    hostname: '127.0.0.1',
    port: 11434,
    path: '/api/generate',
    method: 'POST',
    headers: {
      'Content-Type': 'application/json',
      'Content-Length': Buffer.byteLength(payload)
    }
  };

  const req = http.request(options, (res) => {
    res.setEncoding('utf8');
    res.on('data', (chunk) => {
      try {
        const parsed = JSON.parse(chunk);
        process.stdout.write(parsed.response);
      } catch(err) {
        // Incomplete JSON chunk during transmission stream
      }
    });
  });

  req.write(payload);
  req.end();
}
