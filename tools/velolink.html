How it works
1

You pick a file

Any file, any size. It stays in your browser — nothing is uploaded.

2

A link is generated

VeloLink encodes a WebRTC offer into the URL and waits for the other side to answer via a tiny free signaling relay.

3

Direct browser-to-browser stream

Once connected, data flows at full LAN/WAN speed directly between browsers — no middleman, no size limit.

4

Receiver downloads instantly

The file is reassembled in the receiver's memory and handed to them as a native browser download.
