How does this work?

The Mechanics of URL Storage

1. Parsing Bytes into Text: Standard binary data (like a video file) cannot fit naturally inside a URL string. We encode those raw bytes into a Base64 text string, prefixed by a standard data:video/mp4;base64,... MIME container.

2. Using Hash (#) Routing: Browsers do not transmit anything after the hashtag hash (#) back to web servers. Keeping the payload strictly in the hash guarantees complete zero-knowledge transmission and maintains maximum security.

3. Dynamic Re-Construction: When a recipient opens the VeloLink URL, the browser detects the base64 string, converts it back into a native Blob Object in memory, and points the custom video player to it!
